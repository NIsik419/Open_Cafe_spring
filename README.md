# CAFEFUN

## 프로젝트 소개

이 프로젝트는 사용자들이 다양한 카페를 검색하고 리뷰를 남기며 즐겨찾기할 수 있는 카페 애플리케이션의 백엔드입니다. 네이버 리뷰를 활용한 키워드 분석을 통해 카페별로 세분화된 카테고리를 제공하며, Firebase를 이용한 구글 로그인 기능을 지원합니다.

## 주요 기능

- **카페 정보 관리**
  - 카페 모델을 통해 다양한 카페의 정보를 저장 및 관리
- **사용자 관리**
  - 유저 모델을 통해 사용자 정보 관리
  - 구글 로그인을 통한 인증 (Firebase 사용)
- **리뷰 기능**
  - 사용자들이 카페에 대한 리뷰를 작성하고 조회
- **북마크 기능**
  - 관심 있는 카페를 북마크하여 쉽게 접근
- **방문 기록**
  - 사용자의 카페 방문 기록을 저장 및 관리

## 데이터베이스 구축 과정

1. **데이터 수집**
   - 네이버 리뷰를 크롤링하여 데이터베이스에 저장
2. **키워드 추출 및 분류**
   - 추출된 리뷰에서 키워드를 분석하기 위해 키워드 분류 알고리즘 실행
   - 명사 사전과 형용사 사전을 구분하여 카테고리를 세분화
3. **카페 모델 생성**
   - 추출된 키워드를 기반으로 카페 모델을 설계
   - 각 카페별로 관련 키워드 저장
4. **유저 모델 생성**
   - 사용자 정보를 관리하기 위한 유저 모델 설계
   - 리뷰, 북마크, 방문 기록 등의 기능 연동

## 모델 구조

### 카페 모델 (`Cafe`)

- `id`: 고유 식별자
- `name`: 카페 이름
- `location`: 위치 정보
- `keywords`: 관련 키워드 (명사 및 형용사 사전 기반)
- `reviews`: 리뷰 목록
- `bookmarks`: 북마크된 사용자 목록
- `visits`: 방문 기록

### 유저 모델 (`User`)

- `id`: 고유 식별자
- `googleId`: 구글 계정 ID
- `name`: 사용자 이름
- `email`: 이메일 주소
- `bookmarks`: 북마크한 카페 목록
- `reviews`: 작성한 리뷰 목록
- `visits`: 방문 기록

### 리뷰 모델 (`Review`)

- `id`: 고유 식별자
- `userId`: 작성자 ID
- `cafeId`: 대상 카페 ID
- `content`: 리뷰 내용
- `createdAt`: 작성일

### 북마크 모델 (`Bookmark`)

- `id`: 고유 식별자
- `userId`: 사용자 ID
- `cafeId`: 카페 ID
- `createdAt`: 추가일

### 방문 기록 모델 (`Visit`)

- `id`: 고유 식별자
- `userId`: 사용자 ID
- `cafeId`: 카페 ID
- `visitedAt`: 방문일

## 인증 방식

- **구글 로그인**
  - Firebase Authentication을 사용하여 구글 계정을 통한 인증 제공
  - 사용자 정보는 Firebase를 통해 안전하게 관리

## 사용 기술

- **백엔드 프레임워크**: Node.js / Express.js (예시)
- **데이터베이스**: MongoDB / PostgreSQL (예시)
- **인증**: Firebase Authentication
- **크롤링 및 데이터 처리**: Python (예시)
- **배포**: AWS / Heroku / Firebase Hosting (예시)
